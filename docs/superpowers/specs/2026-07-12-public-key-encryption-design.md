# ZWZ 多接收方公钥加密与可选签名设计

## 目标

为 ZWZ 格式新增一套与现有密码模式互斥的公钥加密模式。发送方可以选择多个接收方公钥创建压缩包，任意接收方都能使用对应私钥解密。发送方还可以选择使用自己的签名身份签名，使接收方能够验证压缩包完整性和来源。

首版必须同时覆盖 ZwzCore 公共 API、CLI 和 GUI，并继续完整支持已有的无密码及密码加密 ZWZ 压缩包。新版压缩包继续使用 `.zwz` 扩展名，由文件头自动识别版本和加密模式。

## 范围

本设计包括：

- ZwZ 应用内生成和管理加密、签名密钥；
- 多接收方公钥加密；
- 可选发送方数字签名；
- 公钥导入和导出；
- 受密码保护的私钥备份和恢复；
- macOS 钥匙串存储与每次私钥使用时的系统身份确认；
- GUI、CLI 和 ZwzCore API；
- 预览、解压、单文件提取、分卷和虚拟磁盘等既有 ZWZ 工作流。

首版不要求兼容 OpenSSL、OpenPGP、age 或其他外部密钥生态。公钥模式与密码模式不能同时用于同一个压缩包。

## 密码学方案

### 身份密钥

每个 ZwZ 身份由两套独立密钥组成：

- X25519 密钥对：用于接收方密钥协商；
- Ed25519 密钥对：用于可选的发送方签名。

每个身份具有用户指定的名称和稳定指纹。指纹定义为带类型和版本前缀的规范化 X25519、Ed25519 公钥字节的 SHA-256 摘要，并以分组十六进制显示。用户必须在密钥管理中手动创建并命名身份；应用不在首次运行时自动生成身份。

### 内容加密

创建公钥加密压缩包时，压缩器为该压缩包生成随机 256 位内容加密密钥。文件数据和索引均使用 AES-256-GCM 认证加密，文件名、路径、大小和目录结构不会以明文存储。

内容按现有 ZWZ 分块模型加密。每个数据块及索引使用唯一 nonce，并将决定解析语义的版本、算法和块标识作为认证数据绑定，防止密文或元数据被替换到其他位置。

### 多接收方密钥封装

压缩器为每个压缩包生成临时 X25519 密钥对。针对每位接收方：

1. 使用临时私钥和接收方 X25519 公钥计算共享秘密；
2. 使用带明确上下文标签的 HKDF-SHA-256 派生密钥封装密钥；
3. 使用 AES-256-GCM 封装同一个内容加密密钥；
4. 写入接收方名称、接收方公钥指纹、临时公钥、封装 nonce、密文和认证标签。

临时密钥每个压缩包重新生成，避免不同压缩包复用同一封装上下文。解压时，应用按指纹查找本机身份并依次尝试匹配的封装记录。某条封装记录损坏时可以继续尝试其他匹配身份；所有可用记录均失败后才终止。

### 可选签名

用户可以选择一个本机身份为压缩包签名，也可以创建未签名压缩包。签名使用该身份的 Ed25519 私钥，覆盖：

- 决定文件格式和算法的文件头字段；
- 完整接收方封装区；
- 所有加密数据块；
- 加密索引及相关认证标签。

压缩包嵌入签名者名称、Ed25519 公钥、指纹和签名值。这样，未导入该公钥的设备也能验证数据未被签名后篡改；若签名公钥已存在于本机联系人库，则额外显示为已知签名者。

验证结果分为：

- 签名有效、已知签名者；
- 签名有效、未知签名者，并显示指纹；
- 未签名；
- 签名无效。

签名无效时默认拒绝预览和解压，不提供将其误认为可信内容的静默回退。

## 文件格式和兼容性

公钥模式使用 ZWZ 格式版本 3，但继续采用 `.zwz` 扩展名。版本 3 避开工作区中已有但尚未接入公开产品路径的实验性版本 2，防止格式编号冲突。文件头明确记录：

- 格式版本；
- 加密模式；
- 内容加密、密钥协商、派生、封装和签名算法标识；
- 接收方数量与封装区位置；
- 签名状态与签名区位置；
- 数据区和加密索引的定位信息。

允许公开的元数据仅包括识别和解封所需的接收方名称、接收方指纹、临时公钥，以及签名者名称、公钥和指纹。文件条目、路径、大小、时间戳和目录结构继续位于加密索引内。

读取端先根据 magic 和版本分派到对应解析器：

- 旧版解析器继续处理现有无密码和密码加密压缩包；
- 新版解析器处理公钥加密、可选签名和新版结构；
- 未知版本返回明确的“不支持的版本”错误。

旧文件无需转换。新写入逻辑不能改变旧版解码行为。

## 密钥管理

### 本机身份

私钥保存在 macOS 钥匙串中。钥匙串项目使用访问控制，确保每次执行私钥解封或签名操作都必须通过 Touch ID 或 Mac 登录密码确认。应用启动或密钥列表读取不能提前解锁私钥，也不提供绕过确认的 CLI 参数。

Touch ID 不可用时允许系统回退到 Mac 登录密码。用户取消确认与没有匹配私钥是两个不同错误。

公钥、名称、指纹和非敏感联系人信息可以直接读取，用于列表、选择和匹配。

### 导入和导出

公钥可以导入和导出为 ZwZ 自有、带版本标识的公钥文件。私钥禁止明文导出。

私钥备份流程必须要求用户设置备份密码。备份容器包含身份元数据、X25519 私钥和 Ed25519 私钥，使用 scrypt（N = 65,536、r = 8、p = 1、随机 16 字节 salt）派生 256 位备份密钥，再由 AES-256-GCM 加密并认证。备份文件显式记录 KDF 算法和参数，以便将来增加更高参数的新版本；读取端不得将未知或过低参数静默降级为直接哈希密码。

导入备份时先验证容器版本、密码和完整性，再写入钥匙串。名称或指纹冲突时要求用户确认，不静默覆盖现有身份。导入成功后，如果当前有因缺少私钥而暂停的解压任务，应自动重新匹配并继续。

删除身份前明确警告：若没有有效备份，使用该身份加密的压缩包可能永久无法恢复。

## 核心组件边界

### ZwzCore

ZwzCore 负责：

- 新版 ZWZ 文件编码和解析；
- 内容密钥生成、分块加密和认证；
- 多接收方密钥封装与解封；
- 签名生成和验证；
- 指纹计算；
- 明确的加密模式、接收方、签名身份和验证结果类型；
- 通过协议获取私钥能力，不直接依赖 GUI 状态。

核心格式和密码学逻辑依赖抽象密钥提供器。生产环境注入钥匙串实现，单元测试注入内存实现，从而在不削弱生产访问控制的情况下稳定测试。

### macOS 密钥存储层

密钥存储层负责：

- 创建、读取、重命名和删除身份；
- 钥匙串访问控制和系统身份确认；
- 公钥联系人库；
- 公钥文件及加密私钥备份文件的导入、导出；
- 冲突处理和非敏感元数据查询。

### GUI 和 CLI

GUI 与 CLI 只编排用户选择和展示结果，不各自实现密码学。两者调用相同的 ZwzCore 和密钥存储能力，保证行为一致。

## 公共 API 设计方向

现有只包含可选 `password` 的调用方式扩展为显式且互斥的加密模式，例如：

- 无加密；
- 密码加密；
- 公钥加密，包含一个或多个接收方以及可选签名身份。

类型系统应阻止调用方同时传入密码和接收方公钥。解密结果应提供结构化签名验证信息，而不是只输出字符串。

保留现有公共 API 兼容入口，并将其映射到无加密或密码模式，使已有调用代码无需立即迁移。新的预览、解压和单文件提取入口可接受密钥提供器，并返回缺少私钥、用户取消、签名无效等具体错误。

## GUI 流程

### 压缩

压缩面板提供“无加密 / 密码 / 公钥”三选一。选择公钥模式后：

- 显示可搜索、可多选的公钥联系人列表；
- 至少选择一个接收方后才能开始；
- 提供可选签名身份选择；
- 使用签名时，在真正访问签名私钥时触发系统身份确认。

### 预览和解压

打开新版压缩包后，应用读取公开的接收方指纹并匹配本机身份。存在匹配项时，在需要解封内容密钥时触发系统确认。

没有匹配私钥时，界面显示压缩包接收方信息，并提供导入私钥备份入口。导入成功后自动返回原任务继续。预览、解压和单文件提取均展示签名状态；签名无效时默认停止。

### 密钥管理

设置中新增密钥管理页，支持：

- 手动创建和命名身份；
- 重命名、查看指纹和删除身份；
- 导入、导出公钥；
- 导出加密私钥备份；
- 从加密备份恢复身份；
- 管理仅含公钥的接收方联系人。

## CLI 流程

CLI 增加密钥创建、列表、重命名、删除、公钥导入导出、私钥备份和恢复命令。具体命令名称在实施计划中与现有命令解析结构对齐。

压缩命令允许重复指定多个接收方，并可选指定签名身份。解压、列表和单文件提取自动匹配钥匙串私钥；没有匹配身份时输出接收方指纹和导入备份的操作提示。

所有涉及私钥的操作保留系统身份确认。非交互环境若无法完成确认，应明确失败，不提供将私钥导出到命令参数或环境变量的替代路径。

## 错误处理和安全要求

系统必须区分并结构化报告：

- 没有匹配私钥；
- 用户取消系统身份确认；
- 私钥读取或钥匙串访问失败；
- 私钥备份密码错误；
- 密钥文件损坏或版本不支持；
- 接收方封装数据损坏；
- 签名无效；
- 压缩包结构损坏；
- 加密内容认证失败。

压缩、备份导出和备份导入均使用同目录临时文件或事务式写入，成功后再原子替换目标，失败或取消后清理半成品。

日志不得记录私钥、内容密钥、共享秘密、派生密钥或备份密码。敏感字节使用完后应在 Swift 和底层库允许的范围内尽快释放或清零。任何 nonce 都不得在同一密钥域内重复。

## 验收与测试

### 密码学和格式

- 多个接收方中的任意一位都能独立解密；未授权身份不能解密。
- 同一组接收方创建不同压缩包时使用不同的临时密钥和封装结果。
- 密码模式与公钥模式严格互斥。
- 篡改文件头、接收方区、数据块、认证标签、索引或签名均被检测。
- 签名、未签名、已知签名者、未知签名者和无效签名状态正确。
- 新格式的单文件、文件夹、空文件、隐藏文件、大文件及分卷往返测试通过。

### 密钥生命周期

- 身份必须由用户手动创建。
- 公钥导入导出后指纹稳定。
- 私钥备份中不存在明文私钥，错误密码和篡改备份均无法导入。
- 恢复后的身份能解密恢复前为其创建的压缩包并生成可验证签名。
- 生产钥匙串实现每次私钥解封和签名都触发系统身份确认；取消操作不会降级为静默访问。

### 产品工作流

- GUI、CLI 和 ZwzCore 对同一压缩包给出一致结果。
- 缺少私钥时可从 GUI 导入备份并自动继续原任务。
- 预览、完整解压、单文件提取、智能解压、虚拟磁盘及分卷流程支持公钥模式。
- 中断或取消不会留下可被误认为有效压缩包或备份的半成品。

### 向后兼容

- 现有无密码 ZWZ 文件继续正常预览和解压。
- 现有密码加密 ZWZ 文件继续正常预览和解压。
- 现有 API 调用无需修改即可维持原行为。
- ZIP 及其他格式行为不受影响。

## 实施边界

实施只围绕本设计所需的格式、密码学、密钥管理和产品入口进行，不重构无关压缩格式或界面。详细文件改动、测试顺序、版本 3 字段布局和迁移步骤将在设计获用户复核后由实施计划确定。
